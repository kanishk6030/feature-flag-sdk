# Contributing to Feature Flag SDK

We love your input! We want to make contributing to Feature Flag SDK as easy and transparent as possible.

## Development Process

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

1. Fork the repo and create your branch from `main`
2. If you've added code, add tests
3. Ensure the test suite passes
4. Issue that pull request!

## Getting Started

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/feature-flag-sdk.git
cd feature-flag-sdk

# Install dependencies
npm install

# Run tests
npm test

# Build
npm run build

# Lint
npm run lint
```

## Pull Request Process

1. Update documentation and tests
2. Ensure all tests pass: `npm test`
3. Update the CHANGELOG.md
4. Describe your changes in the PR description

## Code Style

- Use ESLint to check code style: `npm run lint`
- Write tests for new features
- Keep it simple and readable

## Reporting Bugs

Use GitHub Issues to report bugs. Include:
- Clear description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- SDK version and Node.js version

## Suggesting Enhancements

Use GitHub Issues for enhancement suggestions. Include:
- Clear description of the enhancement
- Rationale
- Possible alternatives
